=== STORAGE ===
there's no D:/ drive, instead it may appear under /mnt/data or /home
/proc and /sys are not on disk, they're virtual


organizing and partitioning data on disk can be done using GPT (guid portioning table) or MBR (master boot record)
MBR: max 2TB 4 partitions... has a TOC at start that tells how partitions are made (/dev/sda,sda1,sda2...)
GPT: >2TB 128 paritions... TOC has copies at different places... gpt is new and better

tools to manage partitions: gparted (gui), parted

everything is a file and folder, usb mounted appears as file or folder, disks appear as file or folders etc.
u can do disk partitioning using fdisk command, select disk from /dev directory

there are different type of file systems:
common ones are ext, xfs, btrfs, dos... ext4 is the best one but might depend on use
after making a partition, we have to format it with a filesystem as partition by default is blank
mkfs.ext4 /dev/sdb1 (e.g. in case where we have made partition of sdb as sdb1)


Partitioning => Formatting => Mounting

mounting: to use a partition, we have to mount it somewhere
mounting means dedicate a partition to some folder
mount /dev/sdb1 /mnt/mydata
umount /mnt/mydata
if we reboot, this will be lost
so add it in /etc/fstab
/dev/sdb1   /mnt/mydata   ext4   defaults   0 2

LVM
Traditional partitions are fixed. You give /home 100GB, it fills up, you're stuck — you can't just "grow" it without unmounting, deleting, recreating, and hoping the adjacent space is free.
LVM adds an abstraction layer between physical disks and the filesystem. You stop thinking in terms of "partitions on disk X" and start thinking in terms of a shared pool of storage you can slice however you want — and resize later.

Physical Disks
     ↓
Physical Volumes (PV)   ← LVM claims these disks/partitions
     ↓
Volume Group (VG)       ← PVs pooled together into one big storage pool
     ↓
Logical Volumes (LV)    ← slices from the pool — these are what you format and mount


RAID (Redundant Array of Independent Disks) combines multiple physical drives into one logical unit
